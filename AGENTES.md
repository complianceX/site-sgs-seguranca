# AGENTS.md — Site Institucional SGS

## Contexto do projeto

Este repositório é o site institucional/marketing do SGS — Sistema de Gestão de Segurança do Trabalho.

Este projeto NÃO é o sistema principal/app operacional do SGS.

O objetivo do site é apresentar o SGS ao mercado, gerar confiança, explicar o produto, captar leads, demonstrar autoridade em Segurança do Trabalho e direcionar visitantes para contato, demonstração ou contratação.

O SGS é um sistema voltado para Segurança do Trabalho, SST, gestão de documentos, APR, DDS, PT, ARR, checklists, inspeções, relatórios fotográficos, auditorias, não conformidades, incidentes e evidências com fotos/vídeos.

Esses módulos devem ser usados apenas como contexto comercial e narrativo do site, não como funcionalidades internas a serem implementadas neste repositório.

## Objetivo visual do site

O site deve transmitir:
- segurança;
- confiança;
- tecnologia;
- autoridade;
- precisão técnica;
- modernidade;
- aparência enterprise;
- clareza comercial;
- credibilidade para empresas, gestores, TSTs e equipes de campo.

## Regras gerais

Antes de alterar qualquer arquivo:
1. Mapear a estrutura do site.
2. Identificar páginas, componentes, estilos, assets, imagens, vídeos e animações.
3. Preservar a identidade do SGS.
4. Não transformar o site em um sistema/app operacional.
5. Não criar telas internas como dashboard, APR, DDS ou login, salvo se já existirem como páginas comerciais.
6. Não mexer em `.env`.
7. Não expor secrets.
8. Não instalar dependências sem justificar.
9. Validar performance, SEO, responsividade e acessibilidade.
10. Priorizar conversão, clareza e confiança.

## Foco dos agentes

Os agentes devem trabalhar em:
- identidade visual;
- hero section;
- textos comerciais;
- chamadas para ação;
- seções institucionais;
- animações leves e profissionais;
- SEO;
- performance;
- responsividade;
- LGPD e privacidade do site;
- formulários de contato;
- prova social;
- diferenciais do produto;
- apresentação dos módulos do SGS como solução comercial.

## Fora do escopo

Não implementar:
- backend do sistema;
- módulos reais de APR, DDS, PT, checklist ou relatórios;
- banco de dados operacional;
- área logada complexa;
- gestão de usuários;
- funcionalidades internas do app SGS.

Se encontrar arquivos relacionados a app/sistema, apenas mapear e avisar antes de alterar.
